# 同舟供应链中心部署更新说明

这份文档用于后续持续开发时，按“本地修改代码 -> 提交到 GitHub -> 服务器拉取更新 -> 构建并重启”的方式发布到线上。

## 当前部署信息

- GitHub 仓库：https://github.com/zuyuanwu-cell/tongzhou-crossborder-supply-center
- 默认分支：`main`
- 服务器项目目录：`/www/wwwroot/gyl.tongzhoukuajing.com`
- 前端构建目录：`/www/wwwroot/gyl.tongzhoukuajing.com/dist`
- 后端进程名：`tongzhou-supply-api`
- 后端端口：`8787`
- Node 版本：服务器可用 `v16.20.2`，项目已兼容 Node 16

## 让 Codex 帮你更新的标准流程

以后你要我改功能时，可以直接说：

```text
请修改 xxx 功能，完成后构建验证、提交并推送到 GitHub，然后告诉我服务器更新命令。
```

我会在本地做这些事：

1. 修改代码。
2. 执行 `npm run build` 验证前端和 TypeScript。
3. 检查 `git status`，不提交 `.env`、`.cache`、`node_modules`、`dist` 和本地素材目录。
4. `git commit` 保存本次修改。
5. `git push origin main` 推送到 GitHub。
6. 告诉你服务器需要执行的更新命令。

## 服务器更新命令

每次我推送 GitHub 后，你在宝塔终端执行：

```bash
cd /www/wwwroot/gyl.tongzhoukuajing.com
git pull origin main
npm ci
npm run build
pm2 restart tongzhou-supply-api --update-env
pm2 save
```

如果宝塔终端找不到 `node`、`npm` 或 `pm2`，先执行：

```bash
export PATH=/www/server/nodejs/v16.20.2/bin:$PATH
```

然后再执行更新命令。

## 更新后检查

执行：

```bash
pm2 list
curl http://127.0.0.1:8787/api/me
```

正常情况：

- `pm2 list` 里 `tongzhou-supply-api` 是 `online`
- `curl` 返回包含 `"ok":true`

然后打开线上网站，刷新页面测试。

## Nginx 配置要点

宝塔网站根目录应指向：

```text
/www/wwwroot/gyl.tongzhoukuajing.com/dist
```

配置里需要保留：

```nginx
location / {
    try_files $uri $uri/ /index.html;
}

location /api/ {
    proxy_pass http://127.0.0.1:8787;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

通常更新代码不需要改 Nginx。

## 环境变量说明

线上 `.env` 位于：

```text
/www/wwwroot/gyl.tongzhoukuajing.com/.env
```

`.env` 不会提交到 GitHub。里面保存接口 Key、登录码、会话密钥等敏感配置。

修改 `.env` 后必须重启后端：

```bash
cd /www/wwwroot/gyl.tongzhoukuajing.com
pm2 restart tongzhou-supply-api --update-env
```

## 常见问题

### 页面能打开，但提示未配置 JIANYUN_API_KEY

检查：

```bash
cd /www/wwwroot/gyl.tongzhoukuajing.com
cat .env
pm2 describe tongzhou-supply-api | grep "exec cwd"
```

确认：

- `.env` 里有真实 `JIANYUN_API_KEY`
- `exec cwd` 是 `/www/wwwroot/gyl.tongzhoukuajing.com`

然后重启：

```bash
pm2 restart tongzhou-supply-api --update-env
```

### 改了管理员密码但旧密码还能登录

默认管理员会被写入：

```text
.cache/users.json
```

如果是新服务器且没有正式用户，可以重置：

```bash
cd /www/wwwroot/gyl.tongzhoukuajing.com
pm2 stop tongzhou-supply-api
rm -f .cache/users.json
pm2 start ecosystem.config.cjs --update-env
pm2 save
```

### GitHub 拉取失败

如果仓库是私有仓库，服务器需要配置 GitHub Token 或 SSH Key。当前如果使用公开仓库，可以直接：

```bash
git pull origin main
```

### npm install 或 npm ci 报 Node 版本问题

项目已兼容 Node 16。先确认服务器使用的是宝塔 Node 16：

```bash
export PATH=/www/server/nodejs/v16.20.2/bin:$PATH
node -v
npm -v
```

## 不要提交到 GitHub 的内容

这些文件或目录不要提交：

- `.env`
- `.cache/`
- `node_modules/`
- `dist/`
- 本地测试视频、临时素材、密钥文件

GitHub 只保存源码和可公开的配置模板。
